# Guide de Contribution

Ce guide décrit les meilleures pratiques pour contribuer à notre projet open source Restalo.

## Prérequis
-Java 21 LTS
- Maven
- Git

## Comment contribuer
1. Forkez le dépôt
2. Créez une branche : `git checkout -b feature/ma-fonctionnalite`
3. Faites vos modifications
4. Commitez avec un message clair : `feat: ajoute nouvelle fonctionnalité`
5. Poussez votre branche
6. Ouvrez une Pull Request

## Bonnes pratiques de développement
- Suivre les conventions de code du projet
- Ajouter des tests pour chaque nouvelle fonctionnalité ou modification
- Mettre à jour la documentation si nécessaire
- Garder les Pull Requests petites et bien documentées

## Processus de revue
- Chaque Pull Request doit être revue par un mainteneur
- Respecter le Code of Conduct
- Une Pull Request ne peut être fusionnée qu'après approbation par un mainteneur

## Modèle utilisé
Le document s’inspire du guide officiel de GitHub : [CONTRIBUTING.md](https://github.com/github/docs/blob/main/CONTRIBUTING.md)

Nous avons choisi ce modèle car :
- Il est clair et structuré
- Il couvre toutes les étapes de contribution
- Il reflète les standards modernes de collaboration en open source
